1. Objective

Investigate why the multi-GPU-container (MGC) regime underperforms native under the updated Exp A3 stable ShareGPT DP=4 benchmark, despite the expectation that a single container owning all GPUs should introduce little or no steady-state serving overhead.

2. Methodology

The redo uses Exp A3's Python harness and the DP=4-only bench subcommand, not the old 2→4→2 cycle or single-shot random benchmark.
